# 24. 滑点控制 (Slippage Control)

## 概念定义

滑点（Slippage）是指预期成交价格与实际成交价格之间的差异。在快速变动的市场中，从下单到成交的时间差可能导致不利的价格变动。

## 滑点产生原因

### 市场因素
```
1. 价格波动
   - 下单瞬间价格已变动
   - 高波动时段尤为明显

2. 流动性不足
   - 买卖价差扩大
   - 订单簿深度不够
   - 大额订单影响市场价格

3. 网络延迟
   - 订单传输时间
   - 交易所撮合时间
   - 行情刷新频率
```

### 订单因素
```
1. 市价单
   - 价格不确定
   - 高滑点风险

2. 大额订单
   - 吃透订单簿多个价位
   - 产生市场冲击

3. 紧急订单
   - 急于成交
   - 接受较差价格
```

## 滑点计算

### 滑点公式
```
绝对滑点 = 实际成交价格 - 预期价格
相对滑点 = (实际成交价格 - 预期价格) / 预期价格 × 100%

多头滑点（买入）：
- 实际成交价格 > 预期价格
- 滑点 = 实际价 - 预期价（正值为不利）

空头滑点（卖出）：
- 实际成交价格 < 预期价格
- 滑点 = 预期价 - 实际价（正值为不利）
```

### 滑点示例
```
买入滑点示例：
预期买入价：100.00元
实际成交：
  200股 @ 100.00
  300股 @ 100.05
  500股 @ 100.10

加权平均成交价：100.06元
绝对滑点：0.06元/股
相对滑点：0.06%
总滑点成本：60元（1000股）
```

## 滑点控制策略

### 1. 订单类型选择
```
优先使用限价单：
✓ 控制成交价格上限
✓ 避免意外滑点
✓ 适合非紧急交易

市价单使用场景：
⚠ 仅用于必须成交的情况
⚠ 接受可能的滑点
⚠ 流动性充足时风险较小
```

### 2. 交易时机选择
```
避开高滑点时段：
❌ 开盘前15分钟
❌ 重大数据发布时
❌ 价格剧烈波动时
❌ 流动性差的时段

选择低滑点时段：
✓ 盘中平稳时段
✓ 成交量活跃时
✓ 价差较小时
```

### 3. 订单拆分执行
```
大额订单拆分：
原订单：买入10,000股 @ 市价

拆分方案：
- 分成10笔，每笔1,000股
- 间隔2-5分钟执行
- 使用限价单逐步买入

优势：
- 减少单次市场冲击
- 平均化成交价格
- 降低滑点成本
```

### 4. 流动性评估
```
下单前检查：
□ 当前买卖价差
   - 价差 < 0.1%：流动性好
   - 价差 0.1-0.3%：一般
   - 价差 > 0.3%：流动性差

□ 订单簿深度
   - 检查目标数量价位的挂单量
   - 是否足够容纳订单

□ 近期成交量
   - 日均成交额是否充足
   - 当前成交量是否异常
```

## 滑点监控与统计

### 滑点记录
```
记录要素：
- 下单时间
- 预期价格
- 实际成交价格
- 成交数量
- 滑点金额/比例
- 市场环境（波动率、流动性）

分析维度：
- 不同时段滑点对比
- 不同标的滑点对比
- 不同订单类型滑点对比
```

### 滑点成本估算
```
交易成本模型：
总成本 = 显性成本 + 隐性成本
显性成本 = 佣金 + 印花税 + 过户费
隐性成本 = 滑点 + 冲击成本 + 机会成本

滑点预算：
- 大盘股：预期滑点 0.05-0.1%
- 中盘股：预期滑点 0.1-0.3%
- 小盘股：预期滑点 0.3-1%
```

## 不同市场滑点特征

| 市场类型 | 滑点水平 | 主要影响因素 |
|----------|----------|--------------|
| 大盘股 | 低 | 流动性极好 |
| 中盘股 | 中 | 流动性一般 |
| 小盘股 | 高 | 流动性差 |
| 期货主力 | 低 | 流动性好 |
| 期货远月 | 高 | 流动性差 |
| 期权 | 高 | 价差大、流动性差 |

## 滑点控制检查清单

```
下单前检查：
□ 当前买卖价差可接受？
□ 订单簿深度足够？
□ 是否处于高波动时段？
□ 是否使用限价单保护？
□ 大额订单是否已拆分？
□ 滑点成本是否在预算内？
```
