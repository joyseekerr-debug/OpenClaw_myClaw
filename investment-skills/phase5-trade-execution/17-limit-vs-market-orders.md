# 17. 限价单与市价单 (Limit vs Market Orders)

## 概念对比

| 维度 | 限价单 (Limit Order) | 市价单 (Market Order) |
|------|----------------------|-----------------------|
| **成交保证** | 不保证 | 保证 |
| **价格保证** | 保证 | 不保证 |
| **成交速度** | 可能较慢 | 立即成交 |
| **滑点风险** | 无 | 有 |
| **适合场景** | 非紧急、控制成本 | 紧急、确保成交 |

## 限价单详解

### 工作原理
```
买入限价单：以指定价格或更低价格买入
卖出限价单：以指定价格或更高价格卖出

订单簿示例：
卖5: 10.10  500手
卖4: 10.08  300手
卖3: 10.05  200手  ← 最佳卖价
卖2: 10.03  100手
卖1: 10.00   50手  ← 最优成交价
─────────────
买1:  9.98  100手  ← 最佳买价
买2:  9.95  200手

若提交买入限价单 10.02元：
- 可以与卖1(10.00)成交
- 无法与卖2(10.03)及更高价格成交
```

### 限价单策略

#### 1. 最优五档即时成交剩余撤销
```
特点：
- 以对手方最优五档价格成交
- 未成交部分自动撤销

适用：
- 希望大部分成交但不愿等待
- 控制平均成本
```

#### 2. 最优五档即时成交剩余转限价
```
特点：
- 最优五档内成交
- 剩余转为限价单继续等待

适用：
- 希望成交但愿意等待更好价格
```

#### 3. 本方最优价格
```
特点：
- 以当前买一/卖一价格申报
- 确保不改善市场

适用：
- 大额订单避免冲击市场
- 被动成交策略
```

### 限价单优缺点

✅ **优点**
- 控制成交价格
- 无滑点风险
- 可作为被动做市策略
- 适合大额分批建仓

❌ **缺点**
- 可能无法成交
- 错失快速行情
- 需要持续盯盘调整
- 极端行情无法离场

## 市价单详解

### 工作原理
```
市价买单：以当前最优卖价立即成交
市价卖单：以当前最优买价立即成交

订单簿示例：
卖1: 10.00   50手
卖2: 10.03  100手
卖3: 10.05  200手

市价买入500手：
- 50手 @ 10.00
- 100手 @ 10.03
- 200手 @ 10.05
- 剩余150手 @ 10.08（下一档）

平均成交价：10.034元
滑点成本：0.034元/股
```

### 市价单适用场景

#### 必须成交的情况
```
1. 止损离场
   - 风控要求必须离场
   - 价格次之，保命第一

2. 套利交易
   - 价差窗口可能迅速关闭
   - 速度优先于价格

3. 期权到期
   - 到期日前必须平仓
   - 避免自动行权风险

4. 强平风险
   - 保证金不足预警
   - 立即减仓避免强平
```

#### 流动性充足时
```
大盘股、活跃期货合约：
- 买卖价差小
- 滑点可控
- 市价单效率高
```

### 市价单风险

⚠️ **闪崩风险**
```
2010年美股闪崩：
- 市价卖单在流动性枯竭时触发
- 价格瞬间下跌99%
- 市价单以极端低价成交
```

⚠️ **流动性风险**
```
小盘股、非活跃时段：
- 买卖价差巨大
- 市价单可能以极劣价格成交
- 建议使用限价单保护
```

## 选择决策树

```
是否需要确保成交？
├── 是 → 使用市价单
│   └── 是否流动性充足？
│       ├── 是 → 直接市价
│       └── 否 → 考虑滑点风险
└── 否 → 使用限价单
    └── 是否急于成交？
        ├── 是 → 限价接近市价
        └── 否 → 限价等待更好价格
```

## 混合策略

### 分层下单
```
大额订单拆分：
- 30% 市价单：确保部分成交
- 70% 限价单：控制平均成本

示例：买入10000股
- 3000股 市价单 立即成交
- 4000股 限价单 当前价
- 3000股 限价单 更低价
```

### 市价转限价
```
部分券商支持：
- 先以市价单提交
- 若未成交或部分成交，转为限价单
- 兼顾速度与价格
```
