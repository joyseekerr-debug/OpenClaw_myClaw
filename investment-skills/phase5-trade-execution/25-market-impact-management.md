# 25. 冲击成本管理 (Market Impact Management)

## 概念定义

冲击成本（Market Impact）是指大额交易对市场价格产生的不利影响。当订单规模超过市场即时流动性时，买入会推高价格，卖出会压低价格，导致实际成交价格劣于预期。

## 冲击成本构成

### 临时冲击 vs 永久冲击
```
临时冲击（Temporary Impact）：
- 交易引起的即时价格波动
- 交易完成后价格恢复
- 由流动性消耗造成

永久冲击（Permanent Impact）：
- 交易后价格持续变化
- 反映信息效应
- 市场解读交易信号

总冲击成本 = 临时冲击 + 永久冲击
```

### 冲击成本公式
```
简化模型：
冲击成本 = η × (订单规模 / 日均成交量)^γ

其中：
η = 市场冲击系数（股票特性）
γ = 弹性系数（通常0.5-0.7）

示例：
- 日均成交量：1000万股
- 订单规模：100万股
- η = 0.5, γ = 0.6

冲击成本 = 0.5 × (100/1000)^0.6 ≈ 0.13 = 13bp
```

## 影响冲击成本的因素

### 订单特征
```
1. 订单规模
   - 规模越大，冲击越大
   - 与日均成交量比值是关键

2. 执行紧迫性
   - 越急迫，冲击越大
   - 时间允许可分散执行

3. 订单方向
   - 买入通常冲击大于卖出
   - 与市场主体方向相关
```

### 市场特征
```
1. 流动性
   - 日均成交量
   - 买卖价差
   - 订单簿深度

2. 波动率
   - 高波动期冲击更大
   - 市场深度下降

3. 市场情绪
   - 恐慌期冲击放大
   - 平静期冲击较小
```

## 冲击成本管理策略

### 1. 订单拆分
```
时间维度拆分（TWAP）：
- 将大单分散到多个时间段
- 均匀执行，隐藏意图
- 适合流动性充足标的

成交量维度拆分（VWAP）：
- 按预期成交量分布执行
- 跟随市场正常交易节奏
- 冲击最小化

价格维度拆分（冰山单）：
- 仅显示部分订单
- 成交后自动补充
- 隐藏真实规模
```

### 2. 智能路由
```
多交易所选择：
- 选择最优价格的交易所
- 分散到多个交易所执行
- 利用交易所间价差

暗池交易（Dark Pool）：
- 场外匿名交易
- 不显示在公开订单簿
- 适合超大额交易
```

### 3. 时机选择
```
低冲击时段：
✓ 盘中流动性充足时
✓ 避免开盘收盘极端时段
✓ 避开重大事件

高冲击时段：
✗ 开盘集合竞价
✗ 重大数据发布时
✗ 市场恐慌期
```

### 4. 被动执行策略
```
做市策略：
- 挂限价单等待成交
- 提供流动性而非消耗
- 可能获得返佣

好处：
- 零冲击或负冲击（获得价差收益）
- 降低交易成本
- 适合不急于成交的订单

坏处：
- 成交不确定
- 需要更长执行时间
- 可能错过行情
```

## 冲击成本预算

### 预交易分析
```
执行前评估：
1. 流动性评估
   - 日均成交量
   - 订单占成交量比例
   - 目标：不超过10-15%

2. 冲击成本预估
   - 使用冲击成本模型
   - 参考历史类似交易
   - 设置冲击成本上限

3. 执行策略选择
   - 根据冲击预估选择算法
   - TWAP/VWAP/冰山单等
```

### 冲击成本上限设定
```
机构常见标准：
- 大盘股：冲击成本 < 5bp
- 中盘股：冲击成本 < 15bp
- 小盘股：冲击成本 < 30bp

超过上限处理：
- 延长执行时间
- 进一步拆分订单
- 考虑替代标的
```

## 冲击成本监控

### 实时监测
```
监测指标：
- 实时成交价格 vs 市场中间价
- 成交量占市场比例
- 价格变动幅度

预警机制：
- 冲击成本接近上限预警
- 自动调整执行速度
- 人工介入决策
```

### 交易后分析
```
分析内容：
- 实际冲击成本 vs 预估
- 执行策略效果评估
- 改进建议

报告要素：
- 订单规模
- 执行时间
- 市场条件
- 实际冲击成本
- 策略评价
```

## 个人投资者启示

### 实际应用
```
1. 控制单笔规模
   - 不超过标的日均成交的1%
   - 小盘股尤其注意

2. 分批建仓
   - 大额资金分多日建仓
   - 避免一次性冲击

3. 使用限价单
   - 控制买入上限价格
   - 避免追高冲击

4. 关注流动性
   - 选择成交量充足的标的
   - 避开流动性枯竭期
```
