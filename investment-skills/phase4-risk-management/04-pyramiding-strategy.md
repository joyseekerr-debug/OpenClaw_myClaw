# 4. 金字塔加仓法 (Pyramiding Position Building)

## 概念定义

金字塔加仓法是一种分批次建仓的策略，随着趋势确认逐步增加仓位，但每次加仓的规模递减，形成"底部大、顶部小"的金字塔形状。这种方法能够在控制风险的同时，让利润充分奔跑。

## 核心原理

### 基本结构
```
        ▲ 第3次加仓（小）
       ▲ ▲ 第2次加仓（中）
      ▲ ▲ ▲ 第1次建仓（大）
     ▲ ▲ ▲ ▲
    ───────────→ 价格
```

### 核心逻辑
1. **首次建仓**：试探性入场，仓位最大但风险可控
2. **趋势确认后加仓**：每次加仓量递减
3. **风险递减**：随着成本上升，新增仓位风险降低
4. **利润锁定**：保留底仓让利润奔跑

## 加仓模型

### 模型1：等比递减（1:2:3 倒金字塔）
```
第1次：3份（30%）
第2次：2份（20%）
第3次：1份（10%）
总计：6份（60%）
```

### 模型2：斐波那契数列
```
第1次：5份
第2次：3份  
第3次：2份
第4次：1份
总计：11份
```

### 模型3：百分比递减
```
每次加仓为前一次的50%-70%
如：100% → 50% → 25% → 12.5%
```

## 加仓触发条件

### 技术触发
- **突破加仓**：价格突破关键阻力位
- **回调加仓**：趋势中健康回调后
- **均线加仓**：价格回踩重要均线
- **成交量确认**：放量突破时

### 时间触发
- **固定间隔**：如每周/每月加仓
- **事件驱动**：财报、政策发布后
- **季节性**：根据历史规律

### 风险触发
- **盈利保护**：浮盈达到X%后才加仓
- **波动率调整**：ATR扩大时暂停加仓

## 应用场景

### 趋势跟踪策略
- 首次在趋势启动时建仓
- 每次创新高/低时加仓
- 最多加仓3-5次

### 价值投资
- 首次在合理估值买入
- 下跌至更低估时加仓
- 注意避免"接飞刀"

### 定投策略
- 定期定额投入
- 市场下跌时额外加仓
- 长期金字塔积累

## 成本与风险控制

### 平均成本计算
```
总成本 = Σ(每次买入数量 × 买入价格)
平均成本 = 总成本 / 总数量
```

### 止损设置
- **统一止损**：全部仓位同一止损价
- **分层止损**：不同批次不同止损
- **移动止损**：随趋势上移止损线

### 最大风险计算
```
最大亏损 = (平均成本 - 止损价) × 总数量
风险占比 = 最大亏损 / 总资金
```

## 正金字塔 vs 倒金字塔

| 类型 | 加仓方式 | 风险特征 | 适用场景 |
|------|----------|----------|----------|
| 正金字塔 | 越加越大 | 风险递增，危险 | 一般不推荐 |
| 倒金字塔 | 越加越小 | 风险递减，安全 | 趋势跟踪 |
| 等比例 | 每次相同 | 风险线性 | 震荡市场 |

⚠️ **警告**：正金字塔加仓（亏损摊平）是高风险行为，容易导致爆仓！

## 注意事项

⚠️ **常见错误**
1. **过早重仓**：首次建仓过重，无加仓空间
2. **频繁加仓**：加仓过密，成本失去优势
3. **亏损加仓**：试图摊平成本，风险累积
4. **无止损**：金字塔变成"埋在塔下"

✅ **最佳实践**
- 首次建仓不超过计划总仓位的40%
- 至少间隔5%-10%的价格才加仓
- 总加仓次数不超过4次
- 设置整体止损，而非逐笔止损

## 实战案例

### 股票趋势跟踪
```
标的：某成长股
计划总仓位：20%资金

第1次：8% @ 50元（突破平台）
第2次：6% @ 55元（新高+10%）
第3次：4% @ 60元（趋势延续）
第4次：2% @ 65元（加速阶段）

平均成本：56.3元
止损价：48元（跌破支撑位）
最大风险：2.4%资金
```
