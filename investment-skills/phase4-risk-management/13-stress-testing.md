# 13. 压力测试 (Stress Testing)

## 概念定义

压力测试（Stress Testing）是通过模拟极端但可能发生的市场情景，评估投资组合在极端条件下的表现和潜在损失的风险管理工具。它补充了VaR等统计方法无法捕捉的尾部风险。

## 压力测试类型

### 1. 历史情景测试
使用过去真实发生的极端事件作为测试情景

**常见历史情景：**
| 事件 | 时间 | 特点 |
|------|------|------|
| 黑色星期一 | 1987年10月 | 美股单日暴跌22% |
| 亚洲金融危机 | 1997年 | 货币大幅贬值 |
| 互联网泡沫破裂 | 2000-2002年 | 科技股崩盘 |
| 次贷危机 | 2008年 | 全球金融危机 |
| 欧债危机 | 2010-2012年 | 主权债务危机 |
| 新冠疫情 | 2020年3月 | 市场闪崩 |

### 2. 假设情景测试
构建理论上可能的极端情景

**假设情景示例：**
```
情景A：利率急升
- 10年期国债收益率上升200bp
- 股市下跌30%
- 信用利差扩大

情景B：地缘政治冲突
- 原油价格暴涨50%
- 全球股市下跌20%
- 避险情绪升温

情景C：流动性危机
- 市场成交量萎缩70%
- 买卖价差扩大3倍
- 无法按市价平仓
```

### 3. 敏感性分析
测试单一风险因素变化的影响
```
股价波动：±10%, ±20%, ±30%
利率变化：±100bp, ±200bp
汇率波动：±5%, ±10%, ±15%
```

## 压力测试实施步骤

### 步骤1：识别风险因素
```
市场风险因素：
- 股票价格
- 利率水平
- 汇率波动
- 商品价格

信用风险因素：
- 违约概率
- 回收率
- 信用利差

流动性风险因素：
- 买卖价差
- 市场深度
- 成交量
```

### 步骤2：构建压力情景
```
单因素压力：
- 某股票价格下跌40%

多因素压力：
- 股票指数下跌35%
- 信用利差扩大300bp
- 波动率指数VIX升至60
```

### 步骤3：计算组合损益
```
方法1：全重估
- 重新计算每个头寸在压力情景下的价值
- 精确但计算量大

方法2：敏感度近似
- 使用希腊字母估计价值变化
- 快速但存在近似误差
```

### 步骤4：分析结果
```
输出指标：
- 组合总损失金额
- 损失占资本金比例
- 最大亏损头寸
- 流动性缺口
```

## 反向压力测试

### 概念
从"什么情景会导致公司破产"出发，逆向推导风险因素组合。

### 应用场景
- 识别隐藏风险点
- 发现模型盲点
- 制定应急预案

### 示例
```
问题：什么情景会导致本公司破产？

逆向推导：
1. 破产条件：资本金耗尽
2. 当前资本金：$10亿
3. 最大可承受损失：$10亿
4. 对应情景：
   - 持有的房地产债券违约率30%
   - 或：股市下跌60%且汇率贬值40%
```

## 压力测试的局限性

⚠️ **局限性**
1. **情景选择主观性**：无法预测所有极端情景
2. **相关性假设**：危机中相关性趋同
3. **静态假设**：未考虑动态对冲
4. **频率问题**：多久测试一次？

✅ **改进方向**
- 定期更新情景库
- 结合专家判断
- 加入流动性压力
- 进行反向压力测试

## 监管要求

### 巴塞尔协议III要求
- 银行必须定期进行压力测试
- 向监管机构报告结果
- 压力测试影响资本要求

### 美联储CCAR
- 年度综合资本分析与审查
- 包括基线、不利、严重不利三种情景
- 未通过测试限制分红和回购
