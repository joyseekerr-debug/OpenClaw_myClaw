# 15. 风险管理体系 (Risk Management System)

## 概念定义

风险管理体系（Risk Management System）是一套完整的组织架构、制度流程、技术工具和人员配置，用于识别、计量、监测和控制投资活动中的各类风险，确保风险水平在可接受范围内。

## 风险管理体系架构

### 三层防线模型
```
┌─────────────────────────────────────────────┐
│  第一层：业务部门                          │
│  - 一线风险识别与控制                      │
│  - 日常风险管理执行                        │
├─────────────────────────────────────────────┤
│  第二层：风险管理部                        │
│  - 风险政策制定                            │
│  - 风险监测与报告                          │
│  - 独立风险评估                            │
├─────────────────────────────────────────────┤
│  第三层：内部审计                          │
│  - 风险管理体系审计                        │
│  - 独立评价与建议                          │
└─────────────────────────────────────────────┘
```

## 风险分类框架

### 市场风险
```
子类别：
├── 利率风险：利率变动导致债券价值波动
├── 汇率风险：外汇敞口价值波动
├── 股票价格风险：持仓股票价值波动
├── 商品价格风险：大宗商品持仓波动
└── 波动率风险：隐含波动率变化

管理工具：
- VaR/CVaR限额
- 止损纪律
- 对冲策略
- 分散化
```

### 信用风险
```
子类别：
├── 违约风险：交易对手无法履约
├── 信用利差风险：信用评级变化
└── 集中风险：单一对手方敞口过大

管理工具：
- 信用评级与限额
- 抵押品管理
- 净额结算协议
- CDS等信用衍生品
```

### 流动性风险
```
子类别：
├── 市场流动性风险：无法按市价交易
├── 资金流动性风险：无法及时获得资金
└── 融资流动性风险：无法展期融资

管理工具：
- 流动性覆盖率(LCR)
- 净稳定资金比率(NSFR)
- 压力测试
- 应急融资计划
```

### 操作风险
```
子类别：
├── 人员风险：内部欺诈、失误
├── 流程风险：流程缺陷、文档错误
├── 系统风险：技术故障、网络攻击
└── 外部事件：自然灾害、监管变化

管理工具：
- 操作手册与检查清单
- 系统冗余与灾备
- 职责分离
- 操作风险资本计提
```

## 风险管理流程

### 1. 风险识别
```
方法：
- 头脑风暴会议
- 历史损失分析
- 情景分析
- 专家访谈

输出：
- 风险清单
- 风险地图
- 关键风险指标(KRI)
```

### 2. 风险计量
```
定量方法：
- VaR/CVaR
- 压力测试
- 敏感性分析
- 蒙特卡洛模拟

定性方法：
- 风险评级
- 专家打分
- 风险矩阵
```

### 3. 风险监测
```
监测内容：
- 风险限额使用情况
- 风险指标变化趋势
- 异常交易行为
- 市场风险因素

监测频率：
- 实时：交易限额、止损
- 日度：VaR、敞口
- 周度：风险报告
- 月度：全面风险评估
```

### 4. 风险控制
```
控制措施：
├── 预防性控制
│   ├── 审批权限设置
│   ├── 交易限额
│   └── 投资白名单
├── 检测性控制
│   ├── 异常交易监控
│   ├── 对账机制
│   └── 风险报告
└── 纠正性控制
    ├── 止损执行
    ├── 应急预案
    └── 危机处理
```

### 5. 风险报告
```
报告层级：
├── 交易员：实时头寸、盈亏
├── 部门经理：日度风险汇总
├── 风险管理部：周度风险报告
├── 高管层：月度风险报告
└── 董事会：季度风险评估

报告内容：
- 风险敞口汇总
- 限额使用情况
- 重大风险事件
- 风险调整收益
```

## 风险管理政策示例

### 交易限额政策
```
1. 头寸限额
   - 单一股票：不超过组合5%
   - 单一行业：不超过组合20%
   - 单一国家：不超过组合30%

2. 止损限额
   - 单笔交易：-5%
   - 单日亏损：-2%组合净值
   - 单月亏损：-10%组合净值

3. VaR限额
   - 日VaR(95%)：不超过组合净值1%
   - 周VaR(95%)：不超过组合净值3%
```

### 例外处理流程
```
情况：交易超出限额

处理流程：
1. 系统自动预警
2. 交易员说明原因
3. 风险经理评估
4. 部门主管审批
5. 限时整改或例外备案
```

## 风险管理文化建设

### 关键要素
```
1. 高层承诺
   - 董事会风险委员会
   - CEO风险管理第一责任人
   - 风险文化自上而下

2. 激励机制
   - 风险调整绩效考核
   - 长期激励为主
   - 合规与风险一票否决

3. 培训教育
   - 入职风险培训
   - 定期风险意识教育
   - 案例分享学习

4. 沟通机制
   - 风险信息共享
   - 鼓励上报风险
   - 无责备报告文化
```

## 个人投资者的风险管理

### 简化版体系
```
1. 投资前
   - 风险评估问卷
   - 确定风险承受能力
   - 设定投资目标和期限

2. 投资中
   - 仓位管理（单笔不超过10%）
   - 止损纪律（单笔-7%，组合-15%）
   - 分散投资（5-10只不同行业）

3. 定期检视
   - 月度回顾盈亏
   - 季度评估策略
   - 年度调整配置

4. 应急处理
   - 黑天鹅应对预案
   - 流动性储备（6个月支出）
   - 保险保障
```
