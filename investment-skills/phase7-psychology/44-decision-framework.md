# 44. 决策框架建立 (Decision Framework)

## 概念定义

决策框架是指一套系统化、结构化的投资决策流程和标准，帮助投资者在复杂信息中做出理性、一致的决策，减少情绪和认知偏差的影响。

## 为什么需要决策框架

### 无框架的问题
```
1. 决策随机
   - 凭感觉买卖
   - 每次标准不同
   - 无法复制成功

2. 情绪干扰
   - 贪婪时追高
   - 恐惧时割肉
   - 后悔时冲动

3. 认知偏差
   - 确认偏误
   - 锚定效应
   - 过度自信

4. 无法复盘
   - 不知道成功/失败原因
   - 无法持续改进
   - 重复同样错误
```

### 有框架的好处
```
✓ 决策一致性
✓ 可复制可学习
✓ 减少情绪影响
✓ 便于复盘改进
✓ 长期期望收益提升
```

## 投资决策框架

### 框架1：投资检查清单

#### 买入检查清单
```
基本面分析：
□ 行业前景是否良好？
□ 公司竞争优势是什么？
□ 财务状况是否健康？
□ 估值是否合理（PE/PB/DCF）？
□ 管理层是否值得信赖？

技术面分析：
□ 趋势方向如何？
□ 是否处于支撑位？
□ 成交量是否配合？
□ 是否有技术买入信号？

风险管理：
□ 最大可接受亏损是多少？
□ 止损位设在哪里？
□ 仓位大小是否合理？
□ 组合相关性如何？

心理检查：
□ 买入是基于分析还是情绪？
□ 是否受到FOMO影响？
□ 是否过度自信？
□ 最坏情况能否接受？
```

#### 卖出检查清单
```
基本面恶化：
□ 行业/公司基本面是否变化？
□ 是否达到目标价？
□ 是否有更好的投资机会？

技术信号：
□ 是否触及止损位？
□ 趋势是否反转？
□ 是否出现卖出信号？

风险管理：
□ 仓位是否过重？
□ 是否需要再平衡？
□ 是否锁定利润？

心理检查：
□ 卖出是基于成本还是价值？
□ 是否被贪婪/恐惧驱动？
□ 若现在空仓会买吗？
```

### 框架3：评分卡模型

```
投资标的评分卡：

维度          权重    评分(1-5)    加权分
─────────────────────────────────────────
行业前景      15%       4          0.60
竞争优势      20%       3          0.60
财务健康      15%       4          0.60
估值吸引力    20%       3          0.60
技术形态      15%       4          0.60
风险可控      15%       3          0.45
─────────────────────────────────────────
总分         100%                 3.45

决策标准：
- 总分≥4.0：强烈买入
- 总分3.5-4.0：买入
- 总分3.0-3.5：观察
- 总分<3.0：放弃
```

### 框架4：决策流程图

```
标准决策流程：

阶段1：信息收集（T-7日到T-1日）
├── 收集基本面信息
├── 分析财务报表
├── 研究行业趋势
├── 技术面分析
└── 输出：分析报告

阶段2：决策准备（T-1日）
├── 完成检查清单
├── 计算目标价/止损价
├── 确定仓位大小
├── 设定买卖条件
└── 输出：交易计划

阶段3：执行（T日）
├── 按计划条件执行
├── 记录成交情况
├── 不临时改变计划
└── 输出：成交记录

阶段4：跟踪（T+1日起）
├── 监控持仓表现
├── 关注止损/止盈条件
├── 定期复盘
└── 输出：跟踪记录

阶段5：退出（满足退出条件）
├── 按计划退出
├── 记录退出原因
├── 计算实际收益
└── 输出：交易总结
```

## 决策框架实施要点

### 1. 严格执行
```
关键：
- 框架的价值在于执行
- 不要临时绕过检查清单
- 不因为"这次不一样"而跳过步骤

例外处理：
- 确实有特殊情况
- 记录例外原因
- 事后评估是否应为例外
```

### 2. 持续改进
```
定期回顾：
- 每月回顾决策质量
- 分析错误决策原因
- 优化框架内容

反馈循环：
- 决策 → 结果 → 分析 → 改进 → 新决策
```

### 3. 个性化定制
```
根据自身调整：
- 投资风格（价值/成长/趋势）
- 风险承受能力
- 时间精力限制
- 专业知识水平

保留核心：
- 风险管理必须严格
- 情绪检查不可少
- 复盘环节必须有
```
