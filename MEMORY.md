## 重要教训

### 阶段性工作汇报原则
**问题:** 2026-02-24 完成 Phase 3 开发后，未及时汇报，导致用户等待19分钟

**根因:** 优先处理技术收尾（文档/提交），而非先告知用户

**规则:**
1. 完成阶段性目标 → **立即汇报** → 再做技术收尾
2. 用户询问进展时 → 先回答状态 → 再解释细节
3. 预计耗时 >10分钟的任务 → 开始前告知，完成后立即汇报

**正确流程:**
```
完成任务 → 通知用户 ✅ → 推送代码/更新文档
```

**避免:**
```
完成任务 → 推送代码/更新文档 → 通知用户 ❌
```
